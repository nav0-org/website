General questions

9 questions
01What is Nav0?

Nav0 is a minimal, privacy-focused web browser built on Electron. The "0" represents zero telemetry, zero data collection, and zero unnecessary features. It's designed for users who want a clean browsing experience without surveillance.

02How is Nav0 different from other browsers?

Unlike mainstream browsers, Nav0:

  • Collects zero telemetry or analytics
  • Has no user accounts or sync services
  • Doesn't phone home to any servers
  • Provides a minimal, distraction-free interface
  • Is 100% open source and auditable
  • Doesn't sell or monetize your data
03Is Nav0 really free?

Yes, completely free. Nav0 is open-source software released under the MIT License. There are no premium features, no paid tiers, no subscriptions, and no advertisements.

04Who develops Nav0?

Nav0 is developed by a small team of privacy-conscious developers and contributors from the open-source community. The project is community-driven and welcomes contributions from anyone who shares the vision.

05What does the "0" in Nav0 mean?

The "0" represents:

  • Zero telemetry
  • Zero tracking
  • Zero data collection
  • Zero bloat
  • Zero distractions

It's a commitment to minimalism in every aspect of the browser.

06Why was Nav0 created?

Nav0 was created in response to the increasing complexity, surveillance, and bloat in modern web browsers. We believe browsing the web should be simple, private, and respectful of user autonomy.

07What platforms does Nav0 support?

Nav0 currently supports:

  • macOS (Intel and Apple Silicon)
  • Windows (10 and 11)
  • Linux (most major distributions)
08Is Nav0 a fork of Chrome or Firefox?

Nav0 is built on top of Electron, which uses the Chromium rendering engine. However, Nav0 is not a fork — it's a custom browser implementation that uses Chromium for web rendering while providing its own minimal UI and privacy-focused architecture.

09What's the current version?

Nav0 is currently in active development. Check our release notes for the latest version. As an early-stage project, we recommend trying it but being aware that some features may still be evolving.

Privacy & security

12 questions
01Does Nav0 really collect no data?

Yes, absolutely zero data. Nav0 doesn't collect:

  • Browsing history (stored locally only)
  • Search queries
  • Usage analytics
  • Crash reports
  • User identifiers
  • IP addresses
  • Any personal information
02How can I verify Nav0's privacy claims?

Nav0 is fully open source. You can:

  • Inspect the source code on GitHub
  • Build the browser from source
  • Use network monitoring tools to verify no data is sent
  • Audit the code for any tracking mechanisms
03Does Nav0 block trackers?

Yes, Nav0 includes built-in tracker blocking using the EasyList filters. This blocks:

  • Advertising trackers
  • Analytics scripts
  • Social media trackers
  • Third-party tracking pixels
04Does Nav0 support fingerprint protection?

Yes, Nav0 implements several fingerprint protection measures:

  • Canvas fingerprint randomization
  • WebGL fingerprint protection
  • Audio context fingerprint protection
  • Font enumeration limits
  • Hardware concurrency standardization
05Is HTTPS enforced?

Yes, Nav0 enforces HTTPS Everywhere by default, automatically upgrading HTTP connections to HTTPS when available.

06How are cookies handled?

Nav0 has strict cookie policies:

  • Third-party cookies are blocked by default
  • Cross-site tracking is prevented
  • SameSite=Strict is enforced where possible
  • Cookies can be cleared on browser exit
07Does Nav0 have a private browsing mode?

Yes, Nav0 includes a private browsing mode where:

  • No history is saved
  • No cookies are persisted
  • No cache is retained
  • Sessions are isolated
08Where is my browsing data stored?

All browsing data is stored locally on your device:

  • macOS: ~/Library/Application Support/nav0/
  • Windows: %APPDATA%\nav0\
  • Linux: ~/.config/nav0/
09Can I use a VPN with Nav0?

Yes, Nav0 works seamlessly with system-level VPNs. We recommend using a VPN for additional privacy, especially on untrusted networks.

10Does Nav0 support Tor?

Currently, Nav0 doesn't have built-in Tor support, but you can configure SOCKS5 proxy settings to route traffic through Tor.

11How are passwords handled?

Nav0 doesn't include a built-in password manager. We recommend using dedicated password managers like:

  • Bitwarden
  • 1Password
  • KeePassXC

This separation of concerns is intentional — password management should be handled by specialized tools.

12What about WebRTC IP leaks?

Nav0 prevents WebRTC IP leaks by:

  • Blocking local IP enumeration
  • Preventing public IP disclosure through WebRTC
  • Disabling unnecessary WebRTC features

Installation

9 questions
01How do I install Nav0?

Visit our installation page for platform-specific instructions:

  • macOS: Download the .dmg or use Homebrew
  • Windows: Download the .exe installer
  • Linux: Use the .AppImage, .deb, or build from source
02What are the system requirements?

Minimum:

  • 4GB RAM
  • 500MB disk space
  • 1GHz processor
  • Internet connection

Recommended:

  • 8GB RAM
  • 1GB disk space
  • 2GHz multi-core processor
03Can I install Nav0 alongside other browsers?

Yes, Nav0 can be installed alongside Chrome, Firefox, Safari, or any other browser without conflicts.

04How do I update Nav0?

Currently, updates are manual. You can:

  • Check our website for new releases
  • Download and install the latest version
  • Watch our GitHub repository for release notifications

We're working on an opt-in update notification system that respects your privacy.

05How do I uninstall Nav0?

macOS: Drag Nav0 from Applications to Trash

Windows: Use "Add or Remove Programs" or run the uninstaller

Linux: Use your package manager or remove the AppImage

To completely remove all data:

# macOS
rm -rf ~/Library/Application\ Support/nav0
# Windows
rmdir /s "%APPDATA%\nav0"
# Linux
rm -rf ~/.config/nav0
06Is Nav0 available on mobile?

Currently, Nav0 is desktop-only. Mobile versions for iOS and Android are planned for the future but not yet available.

07Can I install Nav0 on a USB drive?

Yes, the AppImage version on Linux can run from a USB drive. We're working on portable versions for other platforms.

08Do I need administrator privileges?

Standard installation requires admin privileges, but you can install Nav0 in user space:

  • macOS: Drag to ~/Applications instead of /Applications
  • Windows: Use the portable version
  • Linux: Use AppImage in user directory
09What if my antivirus flags Nav0?

Some antivirus software may flag Nav0 because it's a new, unsigned executable. This is a false positive. You can:

  • Verify the SHA-256 checksum of the download
  • Add Nav0 to your antivirus exceptions
  • Build from source to ensure authenticity

Features & usage

10 questions
01Does Nav0 support extensions?

Currently, Nav0 doesn't support browser extensions. This is intentional — extensions can compromise privacy and add complexity. We provide essential privacy features built-in.

02Can I import bookmarks from other browsers?

Yes, Nav0 supports importing bookmarks from:

  • Chrome / Chromium
  • Firefox
  • Safari
  • HTML bookmark files
03Does Nav0 support tabs?

Yes, Nav0 supports multiple tabs with a clean, minimal tab bar. Features include:

  • Tab pinning
  • Tab muting
  • Tab reordering
  • Recently closed tabs
04What about bookmarks?

Nav0 has a clean bookmark system:

  • Hierarchical folders
  • Quick search
  • Import / export
  • Keyboard shortcuts
05Does Nav0 have a download manager?

Yes, Nav0 includes a built-in download manager with:

  • Progress tracking
  • Pause / resume
  • Download history
  • Custom save locations
06Can I customize the appearance?

Yes, Nav0 supports:

  • Dark / light themes
  • Custom colors
  • Adjustable UI density
  • Custom fonts (limited for fingerprint protection)
07Does Nav0 support keyboard shortcuts?

Yes, Nav0 includes comprehensive keyboard shortcuts. Common ones:

  • Cmd/Ctrl+T — New tab
  • Cmd/Ctrl+W — Close tab
  • Cmd/Ctrl+L — Focus address bar
  • Cmd/Ctrl+R — Reload
  • Cmd/Ctrl+Shift+P — Private window
08Can I sync data between devices?

Nav0 doesn't offer cloud sync to maintain privacy. However, you can:

  • Manually export / import bookmarks
  • Use a self-hosted sync solution
  • Use file synchronization tools (Syncthing, etc.)
09Does Nav0 support PDF viewing?

Yes, Nav0 has a built-in PDF viewer with privacy in mind — no external services are contacted, and PDFs are rendered locally.

10What about web standards support?

Nav0 supports modern web standards through Chromium, including HTML5, CSS3, JavaScript ES2024+, WebGL, and modern Web APIs (with privacy restrictions on sensitive APIs).

Technical questions

10 questions
01What technology stack does Nav0 use?

Nav0 is built with:

  • Electron — Application framework
  • Chromium — Web rendering engine
  • Node.js — Backend runtime
  • TypeScript — Programming language
  • React — UI framework
02Why Electron instead of native?

Electron allows us to:

  • Maintain cross-platform consistency
  • Leverage Chromium's web engine
  • Reduce development overhead
  • Focus on user experience over platform-specific code

While Electron has its drawbacks (memory usage), it enables our small team to deliver a quality browser across platforms.

03How does Nav0 handle memory?

Nav0 implements several memory optimizations:

  • Tab suspension for inactive tabs
  • Aggressive garbage collection
  • Memory limits per tab
  • Minimal background processes
04What about CPU usage?

Nav0 minimizes CPU usage by:

  • Throttling background tabs
  • Disabling unnecessary animations
  • Optimizing rendering pipeline
  • Limiting JavaScript execution in inactive tabs
05Does Nav0 support hardware acceleration?

Yes, hardware acceleration is enabled by default but can be disabled in settings if you experience issues.

06What ports does Nav0 use?

Nav0 uses standard web ports:

  • 80 (HTTP) — Auto-upgraded to HTTPS
  • 443 (HTTPS) — Default secure browsing
  • 53 (DNS) — DNS queries

No additional ports are required.

07Can I use custom DNS?

Yes, Nav0 respects system DNS settings and supports:

  • DNS over HTTPS (DoH)
  • DNS over TLS (DoT)
  • Custom DNS servers
08Does Nav0 support proxies?

Yes, Nav0 supports:

  • HTTP proxies
  • HTTPS proxies
  • SOCKS4/5 proxies
  • PAC file configuration
  • System proxy settings
09What about IPv6?

Yes, Nav0 fully supports IPv6 networks.

10Are there command-line options?

Yes, Nav0 supports command-line flags:

nav0 --help              # Show all options
nav0 --private           # Start in private mode
nav0 --no-extensions     # Disable extensions
nav0 --proxy-server=URL  # Use proxy
nav0 --user-data-dir=DIR # Custom profile directory

Building from source

8 questions
01Why build from source?

Building from source ensures:

  • You're running exactly the code you've reviewed
  • No tampering during distribution
  • Latest features and fixes
  • Custom modifications possible
02What are the build requirements?

You'll need:

  • Node.js 22+
  • npm or pnpm
  • Git
  • Python 3.x (for native modules)
  • Build tools (Xcode on macOS, Visual Studio on Windows, GCC on Linux)
03How long does building take?

Build times vary:

  • First build: 30-60 minutes (downloading dependencies)
  • Subsequent builds: 5-10 minutes
  • Hot reload: Near-instant during development
04Can I build for a different platform?

Cross-compilation is supported but limited:

  • From macOS: Can build for Linux
  • From Linux: Can build for Windows (with Wine)
  • From Windows: Native Windows builds only

For best results, build on the target platform.

05How do I customize the build?

Edit config.json to customize:

  • Default search engine
  • Homepage
  • Privacy settings
  • UI preferences
  • Build flags
06Can I create custom builds?

Yes, the MIT license allows you to:

  • Modify the source code
  • Create branded builds
  • Add or remove features
  • Distribute your version

Just maintain the original copyright notice.

07How do I contribute to development?

To contribute:

  1. Fork the repository
  2. Create a feature branch
  3. Make your changes
  4. Submit a pull request

See our CONTRIBUTING.md for details.

08Is there a development guide?

Yes, comprehensive development documentation is available in the project repository covering setup, architecture, build system, testing, and release processes.

Future plans

8 questions
01What's on the roadmap?

Our public roadmap includes:

  • Mobile versions (iOS, Android)
  • Built-in VPN support
  • Enhanced privacy features

  • Accessibility enhancements

Yes, Nav0 will always be free and open-source. This is a core commitment of the project.

Troubleshooting

8 questions
01Nav0 won't start. What should I do?

Try these steps:

  1. Restart your computer
  2. Check system requirements
  3. Run from terminal to see error messages
  4. Reset Nav0 settings
  5. Reinstall Nav0
02Pages aren't loading correctly

Common solutions:

  • Clear cache and cookies
  • Disable tracker blocking temporarily
  • Check network connection
  • Try a different DNS server
  • Update Nav0
03Nav0 is using too much memory

Reduce memory usage by:

  • Closing unused tabs
  • Enabling tab suspension
  • Disabling hardware acceleration
  • Restarting the browser
  • Checking for memory leaks in extensions (when supported)
04Some websites don't work properly

Privacy features may affect some sites. Try:

  • Disabling fingerprint protection for that site
  • Allowing third-party cookies temporarily
  • Whitelisting the site in tracker blocking
  • Using compatibility mode
05Videos won't play

Video issues can be resolved by:

  • Enabling hardware acceleration
  • Updating graphics drivers
  • Allowing necessary cookies
  • Checking codec support
  • Disabling VPN temporarily
06Downloads are failing

Try:

  • Checking disk space
  • Verifying download permissions
  • Disabling antivirus temporarily
  • Using a different download location
  • Checking firewall settings
07How do I reset Nav0?

To reset all settings:

  1. Close Nav0 completely
  2. Delete the configuration directory (see "Where is my data stored?")
  3. Restart Nav0
08Nav0 crashes frequently

For frequent crashes:

  1. Update to the latest version
  2. Check system logs for errors
  3. Disable hardware acceleration
  4. Run with --disable-gpu
  5. Report the issue with details on GitHub
